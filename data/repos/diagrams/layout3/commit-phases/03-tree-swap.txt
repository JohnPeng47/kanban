COMMIT: THE TREE SWAP
======================
Between mutation and layout phases, the double buffer flips.

Source: packages/react-reconciler/src/ReactFiberWorkLoop.js (commitRoot)


    After mutation, before layout:

          root.current = finishedWork
          ^^^ THE SWAP ^^^

    Before swap:
    +-------------+                  +-------------+
    | current     |   .alternate     | finishedWork|
    | (old state) | <--------------> | (new state, |
    | (what DOM   |                  |  DOM now    |
    |  showed     |                  |  matches)   |
    |  before     |                  |             |
    |  mutation)  |                  |             |
    +-------------+                  +-------------+
          ^
          |
      root.current

    After swap:
    +-------------+                  +-------------+
    | (recycled   |   .alternate     | finishedWork|
    |  for next   | <--------------> | = new       |
    |  render)    |                  |   current   |
    +-------------+                  +-------------+
                                           ^
                                           |
                                       root.current

    WHY BETWEEN MUTATION AND LAYOUT:
    - Layout effects (useLayoutEffect, componentDidMount)
      need to read the "current" tree as the NEW state
    - During mutation, "current" still pointed to old state
      so cleanup functions could read pre-mutation DOM
