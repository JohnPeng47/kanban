COMMIT PHASE: TIMING OVERVIEW
===============================
How all four sub-phases relate to each other and the browser.

Source: packages/react-reconciler/src/ReactFiberWorkLoop.js (commitRoot)


    setState()
       |
       v
    [render phase - interruptible]
       |
       v
    Phase A: beforeMutation
      getSnapshotBeforeUpdate
      (DOM still shows OLD state)
       |
       v
    Phase B: mutation              <-- DOM changes happen here
      appendChild, commitUpdate,
      removeChild
       |
       v
    *** root.current = finishedWork ***   <-- the tree swap
       |
       v
    Phase C: layout                <-- sync effects (before paint)
      useLayoutEffect, componentDidMount,
      ref assignments
       |
       v
    [browser paints]               <-- user sees the update
       |
       v
    Phase D: passive               <-- async effects (after paint)
      useEffect cleanup, then
      useEffect create


    SUBTREE OPTIMIZATION:

    The commit phase does NOT walk the entire tree.
    It uses subtreeFlags (bubbled up during completeWork):

    if (fiber.subtreeFlags & MutationMask) {
      // some descendant has work -- descend
      commitMutationEffects_begin(child);
    }
    if (fiber.flags & MutationMask) {
      // this fiber has work -- do it
      commitMutationEffectsOnFiber(fiber);
    }

    If a subtree has no flags, the entire branch is skipped.
    Commit is O(affected nodes), not O(tree size).
