HOST CONFIG OPERATIONS
======================
During commit phase, the reconciler calls host config functions.
ReactFiberConfigDOM.js implements them as actual DOM operations.
This is the translation layer between React's abstract tree and the browser.

Source: packages/react-dom-bindings/src/client/ReactFiberConfigDOM.js
        packages/react-dom-bindings/src/client/ReactDOMComponent.js


    CREATION:

    createInstance("div", {className:"card"}, ...)
        --> document.createElement("div")
        --> setInitialProperties(el, "div", {className:"card"})
            --> el.className = "card"

    createTextInstance("Card 1", ...)
        --> document.createTextNode("Card 1")


    TREE MANIPULATION:

    appendChild(columnEl, cardEl)
        --> columnEl.appendChild(cardEl)

    insertBefore(columnEl, cardEl, siblingEl)
        --> columnEl.insertBefore(cardEl, siblingEl)

    removeChild(columnEl, cardEl)
        --> columnEl.removeChild(cardEl)

    appendChildToContainer(rootEl, appEl)
        --> rootEl.appendChild(appEl)
        (handles document nodes, comment nodes as mount points)


    UPDATES:

    commitUpdate(cardEl, "div", oldProps, newProps)
        --> updateProperties(cardEl, "div", oldProps, newProps)
            --> diff each prop, apply changes:
                el.className = newProps.className
                el.style.color = newProps.style.color
                el.setAttribute("data-id", newProps["data-id"])

    commitTextUpdate(textNode, "Card 1", "Card 1 (edited)")
        --> textNode.nodeValue = "Card 1 (edited)"


    SPECIAL CASES:

    SVG/MathML:        createInstance uses createElementNS with namespace URI
    Portals:           appendChildToContainer targets a different DOM root
    Controlled inputs: commitUpdate also calls ReactDOMInput to reset .value
    Hoistables:        <link>, <style>, <script> hoist to <head> or <html>
