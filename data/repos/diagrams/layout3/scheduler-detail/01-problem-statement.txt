THE PROBLEM THE SCHEDULER SOLVES
==================================
Why React needs a scheduler at all.

Source: packages/scheduler/src/forks/Scheduler.js


    Browser main thread is single-threaded. It must handle:
      - JavaScript execution
      - Style calculation
      - Layout
      - Paint
      - User input events
      - Animations (rAF)

    If React renders a large tree synchronously (e.g., 10,000 fibers),
    it blocks ALL of the above for the entire duration.

    +------------------------------------------------------------------+
    |                                                                  |
    | WITHOUT SCHEDULER:                                               |
    |                                                                  |
    | |<------------ 200ms render ----------->|<-- browser -->|        |
    | | processing 10,000 fibers              | finally gets  |        |
    | | user clicks are QUEUED                | to paint      |        |
    | | animations FREEZE                     | and handle    |        |
    | | page is UNRESPONSIVE                  | input         |        |
    |                                                                  |
    | WITH SCHEDULER:                                                  |
    |                                                                  |
    | |<5ms>|<brwsr>|<5ms>|<brwsr>|<5ms>|<brwsr>|...|<5ms>|<commit>|  |
    | |work |paint  |work |input  |work |paint  |   |work |done    |  |
    | |     |input  |     |paint  |     |input  |   |     |        |  |
    |                                                                  |
    | Total render time is slightly longer, but the page stays         |
    | responsive throughout.                                           |
    |                                                                  |
    +------------------------------------------------------------------+

    The scheduler breaks React's work into ~5ms chunks, yielding
    between them so the browser can handle input and paint.
