═══ 3. DATA FLOW ═════════════════════════════════════════════════════════════════════════════════════

  WebSocket → board state → KanbanBoard
  ──────────────────────────────────────────────────────────────────────────────────────────────────
  useProjectNavigation
    └─ useRuntimeStateStream ──(WebSocket)──► streamedWorkspaceState
                                                   │
                                                   ▼
  useWorkspaceSync(streamedWorkspaceState) ──► applyWorkspaceState()
    ├─ setBoard(normalized)                         │
    ├─ setSessions(merged)                          ▼
    └─ setCanPersistWorkspaceState(true)       board (useState)
                                                   │
                                                   ├──► KanbanBoard data={board}
                                                   ├──► CardDetailView selection={selectedCard}
                                                   └──► useWorkspacePersistence(board) ──► TRPC save
                                                        [→ workspace-sync-cycle]

  Task start chain: Board drag → TRPC → runtime-api
  ──────────────────────────────────────────────────────────────────────────────────────────────────
  KanbanBoard.onStartTask
    → useTaskStartActions.handleStartTaskFromBoard(taskId)
      → useBoardInteractions.handleStartTask(taskId)
        → useTaskSessions.ensureTaskWorkspace (TRPC workspace.ensureWorktree)
          → useTaskSessions.startTaskSession (TRPC runtime.startTaskSession)
            → runtime-api fork: Cline path or terminal path
                                                        [→ trpc-router-and-api-map]

  Board interactions internal composition
  ──────────────────────────────────────────────────────────────────────────────────────────────────
  useBoardInteractions
    ├─► useLinkedBacklogTaskActions  (auto-start linked backlog tasks on finish)
    └─► useProgrammaticCardMoves    (server-pushed column transitions)
                                                        [→ state-hub-and-registry]

  Project switching → WebSocket reconnect → board hydration
  ──────────────────────────────────────────────────────────────────────────────────────────────────
  useProjectNavigation.handleSelectProject(projectId)
    → onProjectSwitchStart() (resets board, selection, terminal, editor)
    → setRequestedProjectId → new WebSocket → snapshot → useWorkspaceSync hydrates board
                                                        [→ workspace-sync-cycle]


═══ CROSS-REFERENCES ═══════════════════════════════════════════════════════════════════════════════════

  [→ board-task-lifecycle]       Board state schema, column semantics, task CRUD lifecycle
  [→ chat-and-terminal-stacks]   ClineAgentChatPanel and AgentTerminalPanel hook composition
  [→ settings-and-onboarding]    RuntimeSettingsDialog and startup onboarding flow
  [→ workspace-sync-cycle]       Hydration, delta, persistence, and conflict-detection paths
  [→ trpc-router-and-api-map]    TRPC procedures called by useTaskSessions and useGitActions
  [→ state-hub-and-registry]     Server-side WebSocket fan-out and workspace state registry
