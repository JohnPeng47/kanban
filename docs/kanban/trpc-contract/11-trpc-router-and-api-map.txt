═══════════════════════════════════════════════════════════════════════════════════
  SECTION 2 — NAMESPACE CAPABILITY SUMMARY
═══════════════════════════════════════════════════════════════════════════════════

  runtime.*  — the largest namespace, groups into five capability areas:

    TASK SESSIONS        start/stop/send for Cline and terminal sessions + shell
    CLINE CHAT           message CRUD, reload, abort, cancel, slash commands
    CONFIG               global config get/save, debug reset
    PROVIDER & AUTH      provider catalog, models, save, add, update, OAuth,
                         account profile, access gate, Featurebase token
                         [→ cline-provider-and-auth]
    MCP                  auth statuses, OAuth per server, settings load/save
                         [→ cline-provider-and-auth]

  workspace.*  — all require workspace scope (x-kanban-workspace-id header):

    GIT OPS              summary, sync actions, checkout, discard, log, refs, diff
    WORKTREE             ensure / delete task worktrees
    STATE                get/save snapshot, notify updated, workspace changes,
                         task context          [→ state-hub-and-registry]
    SEARCH               fuzzy file search

  workspaceProcedure enforces x-kanban-workspace-id header.
  t.procedure allows calls without a resolved workspace (global config, providers).
